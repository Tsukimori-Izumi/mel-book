# SUP11 機械学習データ管理 アクティビティ詳細

Automotive SPICE 4.0 における SUP.11 機械学習データ管理プロセスのアクティビティ詳細です。

---

## アクティビティ一覧

### SUP11.AC01: MLデータ管理システムの確立
- **目的**: データ収集、ラベリング、アノテーション等のMLデータ管理活動を支援するためのシステムを確立する。
- **対応BP**: SUP11.BP1
- **入力**: なし（組織標準プロセス、インフラ要件）
- **出力**: 16-52 MLデータ管理システム
- **配下タスク**:
  - SUP11.TA0101: データ収集支援システムの検討
  - SUP11.TA0102: データラベリング支援システムの検討
  - SUP11.TA0103: データアノテーション支援システムの検討
  - SUP11.TA0104: 管理システムの確立
  - SUP11.TA0105: 情報源支援の確立
  - SUP11.TA0106: ライフサイクル/ステータスモデルの確立
  - SUP11.TA0107: 関係者窓口の確立

### SUP11.AC02: MLデータの品質アプローチの作成
- **目的**: データの品質基準、バイアス回避策等を含むデータ品質管理のアプローチを定義する。
- **対応BP**: SUP11.BP2
- **入力**: 17-00 要求 (MLデータ要求)
- **出力**: 19-50 MLデータの品質アプローチ
- **配下タスク**:
  - SUP11.TA0201: 品質基準と情報源の関連性定義
  - SUP11.TA0202: 品質基準とラベリングの関連性定義
  - SUP11.TA0203: 品質基準とアノテーションの関連性定義
  - SUP11.TA0204: バイアス回避支援活動の定義
  - SUP11.TA0205: データの品質基準定義
  - SUP11.TA0206: 分析可能性の保証策定義
  - SUP11.TA0207: バイアス回避保証策定義

### SUP11.AC03: MLデータの収集
- **目的**: 生データを収集し、情報源（センサー特性等）の変化を監視する。
- **対応BP**: SUP11.BP3
- **入力**: 19-50 品質アプローチ、16-52 管理システム
- **出力**: 03-53 MLデータ (生データ)
- **配下タスク**:
  - SUP11.TA0301: 生データ情報源の識別
  - SUP11.TA0302: 情報源変化の監視
  - SUP11.TA0303: 生データの収集

### SUP11.AC04: MLデータの処理
- **目的**: 収集したデータを要求に整合するように処理（ラベリング、クレンジング等）する。
- **対応BP**: SUP11.BP4
- **入力**: 03-53 MLデータ (生データ)、17-00 要求
- **出力**: 03-53 MLデータ (処理済データ)
- **配下タスク**:
  - SUP11.TA0401: 処理対象データ情報源の識別
  - SUP11.TA0402: 処理変化の監視
  - SUP11.TA0403: データの処理

### SUP11.AC05: MLデータの品質保証
- **目的**: 処理済データが品質基準を満たしていることを検証し、必要に応じて更新する。
- **対応BP**: SUP11.BP5
- **入力**: 03-53 MLデータ (処理済データ)、19-50 品質アプローチ
- **出力**: 13-18 品質適合性の証拠、15-52 検証結果 (データ品質検証)
- **配下タスク**:
  - SUP11.TA0501: 品質アプローチに従った活動実施
  - SUP11.TA0502: 品質基準満足の保証

### SUP11.AC06: 合意された処理済のMLデータの伝達
- **目的**: 確定したMLデータセットについて関係者と合意し、伝達する。
- **対応BP**: SUP11.BP6
- **入力**: 13-18 品質適合性の証拠
- **出力**: 13-52 情報伝達の証拠
- **配下タスク**:
  - SUP11.TA0601: 関係者への伝達
