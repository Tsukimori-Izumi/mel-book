# スキル設計パターン集

ユースケースに合わせて適切なパターンを選択してください。

---

## パターン1: 順次ワークフロー

**使用場面**: 特定の順序で進む複数ステップのプロセスが必要な時

**構造例**:

```markdown
## ステップ 1: アカウント作成
MCPツール呼び出し: `create_customer`
パラメータ: name, email, company

## ステップ 2: 支払い設定
MCPツール呼び出し: `setup_payment_method`
依存: ステップ1 完了後

## ステップ 3: サブスクリプション作成
MCPツール呼び出し: `create_subscription`
依存: ステップ2 完了後
```

**キーテクニック**:
- 明示的なステップ順序
- ステップ間の依存関係を明記
- 各段階での検証
- 失敗時のロールバック指示

---

## パターン2: マルチMCP連携

**使用場面**: ワークフローが複数のサービスにまたがる時

**構造例**:

```markdown
# フェーズ1: デザインエクスポート（Figma MCP）
Figmaからデザインアセットをエクスポート

# フェーズ2: アセット保存（Drive MCP）
Driveにプロジェクトフォルダを作成し、アセットをアップロード

# フェーズ3: タスク作成（Linear MCP）
開発タスクを作成、アセットリンクを添付

# フェーズ4: 通知（Slack MCP）
#engineering にハンドオフサマリーを投稿
```

**キーテクニック**:
- 明確なフェーズ分離（各MCPごと）
- MCP間のデータ受け渡し
- 次のフェーズに進む前の検証
- 一元的なエラーハンドリング

---

## パターン3: 反復的改善

**使用場面**: 反復によって出力品質が向上する時

**構造例**:

```markdown
# ステップ 1: 初期ドラフト生成
MCPでデータを取得し、最初のドラフトを生成

# ステップ 2: 品質チェック
検証スクリプトを実行: `scripts/check_report.py`
問題を特定（欠落セクション、フォーマット不整合）

# ステップ 3: 改善ループ
特定された各問題に対処し、再検証
終了条件: すべてのチェックに合格

# ステップ 4: 最終化
最終フォーマットを適用し、サマリーを生成
```

**キーテクニック**:
- 明示的な品質基準
- 反復ループの構造化
- 検証スクリプトの活用
- 反復停止条件の明確化

---

## パターン4: コンテキスト認識ツール選択

**使用場面**: 同じ成果を、状況に応じて異なるツールで達成する時

**構造例**:

```markdown
# ステップ 1: コンテキスト分析
ファイルタイプとサイズを確認し、最適な処理ルートを決定

# ルートA: 大容量処理（> 10MB）
クラウドストレージMCP使用

# ルートB: 共同作業（ドキュメント型）
Notion/Docs MCP使用

# ルートC: コード（コードファイル）
GitHub MCP使用

# ステップ 2: コンテキスト説明
選択理由をユーザーに説明
```

**キーテクニック**:
- 明確な判定基準
- フォールバックオプション
- 選択の透明性

---

## パターン5: ドメイン特化インテリジェンス

**使用場面**: ツールアクセス以上の専門知識を追加する時

**構造例**:

```markdown
# ステップ 1: コンプライアンスチェック
MCPで取引詳細を取得し、コンプライアンスルールを適用
- 制裁リスト確認
- 管轄区域許可確認
- リスクレベル評価

# ステップ 2: 条件分岐処理
## 合格の場合:
決済処理MCPツールを呼び出し＋不正チェック

## 不合格の場合:
レビュー用にフラグ＋コンプライアンスケースを作成

# ステップ 3: 監査証跡
すべてのチェックをログ、処理決定を記録
```

**キーテクニック**:
- ロジックに埋め込まれたドメイン専門知識
- アクション前のコンプライアンス確認
- 包括的な文書化
- 明確なガバナンス

---

## パターン選択ガイド

| 要件 | 推奨パターン |
|------|------------|
| 決まった手順のプロセス | パターン1: 順次ワークフロー |
| 複数サービスの連携 | パターン2: マルチMCP連携 |
| 品質が重要な出力 | パターン3: 反復的改善 |
| 条件分岐が多い | パターン4: コンテキスト認識 |
| 専門的な判断が必要 | パターン5: ドメイン特化 |
| 複数パターンの組み合わせ | 上記を組み合わせたハイブリッド |
