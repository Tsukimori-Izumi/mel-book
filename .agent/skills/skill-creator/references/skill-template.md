# スキルテンプレート

新しいスキルを作成する際のコピー＆ペースト用テンプレートです。  
角括弧 `[ ]` の部分を具体的な内容に置き換えてください。

## フォルダ構造テンプレート

```
[スキル名]/              ← kebab-case必須
├── SKILL.md             ← 必須：メインスキルファイル
├── scripts/             ← 任意：実行可能なコード
│   └── [スクリプト名].py
├── references/          ← 任意：参照ドキュメント
│   └── [参考資料名].md
└── assets/              ← 任意：テンプレート等
    └── [テンプレート名].md
```

## description フィールドの書き方ガイド

### 必須要素

1. **何をするか** — スキルの機能を明確に
2. **いつ使うか** — トリガーフレーズを具体的に

### 良い例

```yaml
description: |
  Linearプロジェクトワークフローを管理。スプリント計画、タスク作成、
  ステータス追跡を行う。「スプリント」「Linearタスク」「プロジェクト計画」
  「チケットを作成して」と言われた時に使用する。
```

### 制約

- 1024文字以下
- XMLタグ（`<` `>`）禁止
- 名前に「claude」「anthropic」を含めない
