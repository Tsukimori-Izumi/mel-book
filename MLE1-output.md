## AC01: 機械学習要求の取得と分解

### TA0101: 機械学習要件の取得

上位要求 `req1` から機械学習に関連する要件を抽出する。

FizzBuzzは本来ルールベースの問題だが、MLE-bookプロセスでは**機械学習アプローチ**で開発する。
したがって、FizzBuzzのルールを機械学習モデルに学習させるアプローチとして要件を定義する。

| ML要件ID | 上位要求ID | ML要件内容 | 種別 |
|---|---|---|---|
| MLR-001 | req1 | 入力（1〜100の整数）に対して正しいFizzBuzz出力を予測するMLモデルを構築する | 機能 |
| MLR-002 | req1 | FizzBuzzルール：3の倍数→"Fizz"、5の倍数→"Buzz"、15の倍数→"FizzBuzz"、それ以外→数値そのまま | 機能 |
| MLR-003 | req1 | モデルの予測精度は100%を目標とする（ルールベースの代替のため） | 非機能 |
| MLR-004 | req1 | 学習データとして1〜100の整数と正解ラベルのペアを用意する | データ |
| MLR-005 | req1 | 推論結果をコンソールに出力する | 機能 |

### TA0102: 機械学習要件の分解

各ML要件をさらに実装可能な粒度に分解する。

**MLR-001 の分解:**
| サブID | 内容 |
|---|---|
| MLR-001-a | 入力: 整数値（1〜100） |
| MLR-001-b | 出力: 4クラス分類（"Fizz", "Buzz", "FizzBuzz", "数値"） |
| MLR-001-c | 分類問題としてモデルを構築する |

**MLR-002 の分解:**
| サブID | 内容 |
|---|---|
| MLR-002-a | 3の倍数判定: input % 3 == 0 → "Fizz" |
| MLR-002-b | 5の倍数判定: input % 5 == 0 → "Buzz" |
| MLR-002-c | 15の倍数判定: input % 15 == 0 → "FizzBuzz" |
| MLR-002-d | その他: 数値をそのまま出力 |

**MLR-003 の分解:**
| サブID | 内容 |
|---|---|
| MLR-003-a | トレーニングデータに対する精度100% |
| MLR-003-b | テストデータ（未知の入力）に対する精度100% |

**MLR-004 の分解:**
| サブID | 内容 |
|---|---|
| MLR-004-a | 入力特徴量: 整数値、mod 3、mod 5 |
| MLR-004-b | ラベル: "Fizz"=0, "Buzz"=1, "FizzBuzz"=2, "数値"=3 |
| MLR-004-c | データ数: 100件（1〜100） |

**MLR-005 の分解:**
| サブID | 内容 |
|---|---|
| MLR-005-a | 1〜100の各整数に対して推論結果を表示する |
| MLR-005-b | 出力形式: "[数値]: [予測結果]" |

## AC02: 実現・検証可能性の検証と統合

### TA0201: 運用環境への影響

| 項目 | 内容 |
|---|---|
| デプロイ先 | ローカルPC（コンソールアプリケーション） |
| 処理能力 | 一般的なPC（CPU推論で十分） |
| メモリ | 最小限（100件のデータ＋軽量モデル） |
| 環境制約 | Python実行環境が必要 |
| 影響評価 | **低** — 軽量な問題のため運用環境への影響は最小 |

### TA0202: 実現方法の可能性

| アプローチ | 実現性 | リスク | 備考 |
|---|---|---|---|
| ニューラルネットワーク（MLP） | ◎ 高い | 低 | シンプルな分類問題に適合 |
| 決定木 | ◎ 高い | 低 | ルールベース的な問題に適合 |
| SVM | ○ 中程度 | 低 | 分類可能だがオーバーキル |
| ロジスティック回帰 | △ 低い | 中 | 非線形のmod演算に不向き |

**推奨**: ニューラルネットワーク（MLP）または決定木

### TA0203: 検証方法の可能性

| 検証項目 | 検証方法 | 評価指標 |
|---|---|---|
| 予測精度 | 全データ（1〜100）での正答率 | Accuracy = 100% |
| 汎化性能 | 101〜200の範囲外データでのテスト | Accuracy |
| ルール適合性 | 各クラス（Fizz/Buzz/FizzBuzz/数値）ごとの正答確認 | クラス別Accuracy |

### TA0204: 機械学習要件の合成

統合された要件セット：

1. **入力**: 整数値を特徴量に変換（値そのもの、mod 3、mod 5）
2. **モデル**: 4クラス分類器（MLP推奨）
3. **学習**: 1〜100の教師データで学習
4. **評価**: 全データで精度100%を達成
5. **出力**: コンソールに "[数値]: [予測結果]" 形式で表示

矛盾する要件なし。依存関係: MLR-004 → MLR-001 → MLR-005

### TA0205: 優先順位付け

| ML要件ID | 要件内容 | 優先度 | 根拠 |
|---|---|---|---|
| MLR-004 | 学習データの準備 | **1（最高）** | 他のすべての要件の前提条件 |
| MLR-001 | FizzBuzz予測MLモデル構築 | **2（高）** | コア機能 |
| MLR-002 | FizzBuzzルール定義 | **2（高）** | MLR-004のラベル定義に必要 |
| MLR-003 | 予測精度100%目標 | **3（中）** | 品質基準 |
| MLR-005 | コンソール出力 | **4（低）** | モデル完成後に対応 |
