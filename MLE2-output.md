## AC01: MLアーキテクチャを作成する

### TA0101: 前処理・処理・後処理の仕様化

```mermaid
flowchart LR
  subgraph 前処理
    A[整数入力] --> B[特徴量変換]
    B --> C["[値, 値%3, 値%5]"]
  end
  subgraph 処理
    C --> D[MLPモデル]
    D --> E[4クラス確率]
  end
  subgraph 後処理
    E --> F[argmax]
    F --> G["ラベル変換"]
    G --> H["出力文字列"]
  end
```

| パイプライン段階 | 仕様 |
|---|---|
| **前処理** | 入力整数 → 特徴量ベクトル `[x, x%3, x%5]` に変換 |
| **処理** | MLP分類器で4クラス分類（Fizz/Buzz/FizzBuzz/数値） |
| **後処理** | 予測クラスIDをラベル文字列に変換し出力 |

### TA0102: MLアーキテクチャコンポーネントの検討

| コンポーネント | 責務 | 技術選定 |
|---|---|---|
| **DataGenerator** | 学習データ生成（1〜100のFizzBuzzペア） | NumPy |
| **FeatureExtractor** | 特徴量変換 `[x, x%3, x%5]` | NumPy |
| **MLPClassifier** | 4クラス分類モデル | scikit-learn MLPClassifier |
| **LabelDecoder** | クラスID → 文字列変換 | Python dict |
| **OutputFormatter** | 結果のコンソール出力 | Python print |

### TA0103: ハイパーパラメータの仕様化

| パラメータ | 値 | 説明 |
|---|---|---|
| hidden_layer_sizes | (64, 32) | 隠れ層：64ユニット→32ユニット |
| activation | relu | 活性化関数 |
| solver | adam | 最適化アルゴリズム |
| max_iter | 2000 | 最大反復回数 |
| random_state | 42 | 再現性のための乱数シード |
| learning_rate_init | 0.001 | 初期学習率 |

### TA0104: MLアーキテクチャエレメントの仕様化

```
入力層: 3ユニット（x, x%3, x%5）
  ↓
隠れ層1: 64ユニット（ReLU）
  ↓
隠れ層2: 32ユニット（ReLU）
  ↓
出力層: 4ユニット（Softmax）
  → クラス0: Fizz
  → クラス1: Buzz
  → クラス2: FizzBuzz
  → クラス3: 数値そのまま
```

## AC03: エレメント分析

### TA0301: 分析基準の定義

| 基準 | 評価方法 | 合否判定 |
|---|---|---|
| 信頼性 | 全データで正答率100%達成 | Accuracy = 100% |
| 説明可能性 | 特徴量の意味が解釈可能 | mod演算は解釈可能 |
| スケーラビリティ | 任意の範囲に拡張可能 | 特徴量設計が汎用的か |
| メンテナビリティ | コードの可読性・保守性 | scikit-learn標準API使用 |

### TA0302: エレメント分析結果

| コンポーネント | 信頼性 | 説明可能性 | スケーラビリティ | メンテナビリティ |
|---|---|---|---|---|
| DataGenerator | ✅ | ✅ | ✅ | ✅ |
| FeatureExtractor | ✅ | ✅ 高い（mod演算は数学的に明確） | ✅ | ✅ |
| MLPClassifier | ⚠️ 要検証 | △ ブラックボックス | ✅ | ✅ |
| LabelDecoder | ✅ | ✅ | ✅ | ✅ |
| OutputFormatter | ✅ | ✅ | ✅ | ✅ |

**リスク**: MLPは十分な学習で100%精度を達成できるか要検証  
**改善提案**: 特徴量に `x%15` を追加することでFizzBuzz判定の精度向上が見込める

→ **特徴量を `[x, x%3, x%5, x%15]` に更新**
