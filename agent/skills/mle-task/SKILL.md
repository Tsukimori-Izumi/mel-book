# MLE-book タスク実行ガイド

MLE-bookドメインの**タスク（TA）**を実行するためのガイドスキルです。  
タスクはアクティビティ（AC）の配下にあり、最も具体的な作業単位です。

---

## ドメイン階層

```
プロセス (MLE1, MLE2, ...)
  └── アクティビティ (AC)
       └── タスク (TA)  ← このスキルが対象
```

## タスクID体系

```
MLE[プロセス番号].TA[アクティビティ番号2桁][タスク番号2桁]
例: MLE1.TA0102 = MLE1プロセス → AC01 → 2番目のタスク
```

---

## タスク実行ワークフロー

### ステップ 1: 対象タスクの特定

ユーザーに以下を確認する：
1. **タスクID**: 例 `MLE1.TA0101`
2. **前提条件**: 前工程のタスクが完了しているか

### ステップ 2: タスク情報の参照

対象プロセスのタスク詳細リファレンスを参照する：
- MLE1の場合: `references/mle1-tasks.md` を参照
- MLE2の場合: `references/mle2-tasks.md` を参照

以下の情報を確認する：
- **所属アクティビティ**: 親アクティビティの情報
- **タスク内容**: 作業の概要
- **具体的な手順**: ステップバイステップの実施手順
- **入力**: 必要な入力情報
- **出力**: 期待される出力成果物

### ステップ 3: タスクの実行

リファレンスに記載された手順に従い、タスクを実行する。  
実行中は以下を意識する：
- 入力成果物が揃っているか
- 出力成果物の品質が十分か
- 次のタスクへの引き継ぎ事項があるか

### ステップ 4: 完了報告

- [ ] タスクの出力成果物が生成されたか
- [ ] 品質が要件を満たしているか
- [ ] 次のタスクまたはアクティビティへの準備ができたか
